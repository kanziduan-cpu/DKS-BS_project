```
Android App
    ├─ 本地数据库 (Room)
    │   ├─ 设备数据
    │   ├─ 场景数据
    │   └─ 临时传感器数据
    │

    │   └─ 实时消息推送
    │
    └─ Supabase云端存储
        ├─ 传感器数据表 (sensor_data)
        └─ 统计数据表 (sensor_data_stats)
```

## 🔧 云端服务器配置

### 1. 服务器环境

#### 系统要求
- **操作系统**：Ubuntu 22.04 LTS或更高版本
- **内存**：至少2GB RAM
- **存储**：至少20GB SSD
- **网络**：公网IP，开放端口3001和1883

#### 必装软件

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 安装PM2
sudo npm install -g pm2

# 安装MQTT服务器
sudo apt install -y mosquitto mosquitto-clients

# 安装Nginx（可选）
sudo apt install -y nginx
```

### 2. 端口配置

#### 需要开放的端口

| 端口 | 用途 | 协议 |
|------|------|------|
| 3001 | HTTP API服务 | TCP |
| 1883 | MQTT消息服务 | TCP |
| 80 | HTTP (可选) | TCP |
| 443 | HTTPS (可选) | TCP |

#### 防火墙配置

```bash
# 使用UFW
sudo ufw allow 3001/tcp
sudo ufw allow 1883/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# 验证
sudo ufw status
```

### 3. API服务部署

#### 上传项目文件
```bash
# 上传cloud-server目录到服务器
scp -r cloud-server/ root@your-server-ip:/opt/warehouse-monitor/

# SSH连接到服务器
ssh root@your-server-ip
```

#### 安装依赖
```bash
cd /opt/warehouse-monitor/cloud-server
npm install
```

#### 配置环境变量
```bash
# 创建.env文件
cat > .env << 'EOF'
# 服务器配置
PORT=3001
NODE_ENV=production

# 数据库配置
# 如果使用Supabase，填入Supabase凭据
SUPABASE_URL=your-supabase-url
SUPABASE_ANON_KEY=your-supabase-anon-key

# MQTT配置
MQTT_HOST=localhost
MQTT_PORT=1883
MQTT_USERNAME=testuser
MQTT_PASSWORD=123456

# 其他配置
LOG_LEVEL=info
EOF
```

#### 启动服务
```bash
# 使用PM2启动
pm2 start server.js --name warehouse-monitor

# 设置开机自启
pm2 startup
pm2 save

# 查看状态
pm2 status
pm2 logs warehouse-monitor
```

### 4. MQTT服务配置

#### 配置Mosquitto

```bash
# 创建配置文件
sudo nano /etc/mosquitto/mosquitto.conf
```

添加以下内容：
```ini
listener 1883
protocol mqtt
allow_anonymous true
log_dest file /var/log/mosquitto/mosquitto.log
connection_messages true
log_timestamp true
```

#### 启动MQTT服务
```bash
sudo systemctl enable mosquitto
sudo systemctl start mosquitto
sudo systemctl status mosquitto
```

### 5. Supabase数据库配置

#### 创建Supabase项目

1. 访问 https://supabase.com
2. 注册并创建新项目
3. 选择离你较近的区域
4. 记录项目URL和API密钥

#### 创建数据库表

在Supabase的SQL Editor中执行以下SQL：

```sql
-- 删除现有表（如果存在）
DROP TABLE IF EXISTS sensor_data CASCADE;
DROP TABLE IF EXISTS sensor_data_stats CASCADE;

-- 创建传感器数据表
CREATE TABLE sensor_data (
    id BIGSERIAL PRIMARY KEY,
    device_id TEXT NOT NULL,
    machine_code TEXT,
    temp NUMERIC(6, 2),
    hum NUMERIC(6, 2),
    received_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建索引
CREATE INDEX idx_sensor_data_device_time 
ON sensor_data(device_id, received_at DESC);

-- 开启RLS
ALTER TABLE sensor_data ENABLE ROW LEVEL SECURITY;

-- 添加策略
CREATE POLICY "Enable insert for anonymous users" 
ON sensor_data FOR INSERT WITH CHECK (true);

CREATE POLICY "Enable read for anonymous users" 
ON sensor_data FOR SELECT USING (true);

-- 创建统计表
CREATE TABLE sensor_data_stats (
    id BIGSERIAL PRIMARY KEY,
    device_id TEXT NOT NULL,
    window_start TIMESTAMP WITH TIME ZONE NOT NULL, 
    window_end TIMESTAMP WITH TIME ZONE NOT NULL, 
    temp_avg NUMERIC(6, 2),      
    temp_max NUMERIC(6, 2),      
    temp_min NUMERIC(6, 2),      
    hum_avg NUMERIC(6, 2),       
    hum_max NUMERIC(6, 2),       
    hum_min NUMERIC(6, 2),       
    sample_count INT DEFAULT 0,  
    discarded_count INT DEFAULT 0, 
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_stats_device_time 
ON sensor_data_stats(device_id, window_start DESC);

ALTER TABLE sensor_data_stats ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Enable read stats for anonymous" 
ON sensor_data_stats FOR SELECT USING (true);
```

#### 配置Row Level Security (RLS)

```sql
-- 为匿名用户启用插入权限
CREATE POLICY "Allow anonymous insert" ON sensor_data
FOR INSERT WITH CHECK (true);

-- 为匿名用户启用查询权限
CREATE POLICY "Allow anonymous select" ON sensor_data
FOR SELECT USING (true);
```

## 📱 Android应用配置

### 1. 修改服务器配置

#### API服务配置

编辑文件：`app/src/main/java/com/warehouse/monitor/network/ApiService.java`

```java
String BASE_URL = "http://your-server-ip:3001/api/";
```

#### MQTT配置

编辑文件：`app/src/main/java/com/warehouse/monitor/mqtt/MqttConfig.java`

```java
public static final String SERVER_HOST = "your-server-ip";
public static final int SERVER_PORT = 1883;
```

#### Supabase配置

编辑文件：`app/src/main/java/com/warehouse/monitor/network/SupabaseClient.java`

```java
private static final String SUPABASE_URL = "https://your-project.supabase.co";
private static final String SUPABASE_ANON_KEY = "your-anon-key";
```

### 2. 添加网络权限

编辑文件：`app/src/main/AndroidManifest.xml`

```xml
<!-- 网络权限 -->
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
<uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />
```

### 3. 配置网络安全

编辑文件：`app/src/main/res/xml/network_security_config.xml`

```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>
```

## 🧪 测试流程

### 1. 服务器端测试

#### 测试API服务
```bash
# 测试API端点
curl http://your-server-ip:3001/api/

# 测试登录接口
curl -X POST http://your-server-ip:3001/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456"}'
```

#### 测试MQTT服务
```bash
# 订阅主题
mosquitto_sub -h localhost -p 1883 -u testuser -P 123456 -t "sensor/#" -v

# 发布测试消息
mosquitto_pub -h localhost -p 1883 -u testuser -P 123456 \
  -t "sensor/data" \
  -m '{"temperature":25.5,"humidity":60.2}'
```

#### 测试Supabase连接
```bash
# 使用Supabase客户端测试
# 在cloud-server目录下
node -e "
const { createClient } = require('@supabase/supabase-js');
const supabase = createClient('YOUR_URL', 'YOUR_KEY');
supabase.from('sensor_data').select('count').limit(1)
  .then(({ data, error }) => console.log('连接成功:', data))
  .catch(err => console.log('连接失败:', err));
"
```

## 🚨 常见问题解决

### 问题1：无法连接到服务器

**检查**：
- 服务器IP地址是否正确
- 防火墙是否开放相应端口
- 网络连接是否正常

**解决**：
```bash
# 测试网络连接
ping your-server-ip

# 测试端口
telnet your-server-ip 3001
telnet your-server-ip 1883

# 检查防火墙
sudo ufw status
```

### 问题2：Supabase连接失败

**检查**：
- Supabase URL和密钥是否正确
- RLS策略是否正确配置
- 表结构是否正确

**解决**：
```bash
# 在Supabase控制台中
1. 检查API密钥是否有效
2. 检查RLS策略是否允许访问
3. 查看SQL执行日志
```

### 问题3：MQTT连接超时

**检查**：
- MQTT服务是否运行
- 端口1883是否开放
- 用户名密码是否正确

**解决**：
```bash
# 检查MQTT服务
systemctl status mosquitto

# 检查端口
netstat -tlnp | grep 1883

# 重启MQTT服务
systemctl restart mosquitto
```
