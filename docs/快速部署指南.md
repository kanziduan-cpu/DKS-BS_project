## 📦 部署步骤

### 步骤1: 连接到服务器

```bash
# Windows用户使用PowerShell或Git Bash
ssh root@43.99.24.178

# 输入密码后登录
```

### 步骤2: 创建应用目录

```bash
mkdir -p /opt/warehouse-monitor
cd /opt/warehouse-monitor
```

### 步骤3: 安装Node.js和npm

```bash
# 添加Node.js仓库
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
apt-get install -y nodejs

# 验证安装
node -v
npm -v
```

### 步骤4: 上传代码文件

**在本地电脑执行：**

```bash
# 上传cloud-server目录下所有文件
scp cloud-server/* root@43.99.24.178:/opt/warehouse-monitor/
```

或使用WinSCP、FileZilla等图形化工具上传。

### 步骤5: 安装依赖并启动

```bash
# 在服务器上执行
cd /opt/warehouse-monitor
npm install

# 启动服务（测试模式）
node server.js
```

看到以下输出说明启动成功：
```
数据库表初始化完成
MQTT Broker运行在端口 1883
HTTP服务器运行在端口 3000
WebSocket服务器运行在端口 3001
云端服务器已启动
```

### 步骤6: 使用PM2管理进程（推荐）

```bash
# 安装PM2
npm install -g pm2

# 启动服务
pm2 start server.js --name warehouse-server

# 设置开机自启
pm2 startup systemd -u root --hp /root
pm2 save
```

### 步骤7: 配置阿里云安全组

**重要步骤！否则无法外网访问！**

1. 登录 [阿里云ECS控制台](https://ecs.console.aliyun.com/)
2. 找到实例 `iZj6c5292t3vfth81nfg4aZ`
3. 点击"安全组" → 进入安全组配置
4. 添加入方向规则：

| 优先级 | 协议类型 | 端口范围 | 授权对象 | 描述 |
|--------|---------|---------|---------|------|
| 100 | TCP | 1883/1883 | 0.0.0.0/0 | MQTT通信 |
| 101 | TCP | 3000/3000 | 0.0.0.0/0 | HTTP API |
| 102 | TCP | 3001/3001 | 0.0.0.0/0 | WebSocket |
| 200 | TCP | 22/22 | 你的IP地址 | SSH管理 |

## 🐛 故障排查

### 问题1: 无法连接到服务器

**检查项：**
1. 服务器是否正常运行: `pm2 status`
2. 防火墙是否开放端口: `ufw status`
3. 阿里云安全组是否配置正确

### 问题2: 端口不通

**排查步骤：**
```bash
# 在服务器内部测试
telnet 127.0.0.1 1883

# 查看端口监听
netstat -tlnp | grep -E '1883|3000|3001'

# 查看防火墙规则
iptables -L -n
```

### 问题3: APP连接超时

**检查项：**
1. 手机网络是否正常
2. 服务器地址配置是否正确
3. 是否在局域网环境（尝试切换到4G/5G）
4. 安全组是否已开放

### 问题4: 数据库错误

**解决方案：**
```bash
# 删除数据库文件
rm /opt/warehouse-monitor/data/warehouse.db

# 重启服务
pm2 restart warehouse-server
```

## 🔒 安全建议

1. **修改SSH端口**
   ```bash
   nano /etc/ssh/sshd_config
   # 修改 Port 22 为其他端口
   service sshd restart
   ```

2. **禁用root远程登录**
   ```bash
   # 创建普通用户
   adduser yourname
   usermod -aG sudo yourname
   
   # 禁用root登录
   nano /etc/ssh/sshd_config
   # 修改 PermitRootLogin no
   ```

3. **配置防火墙**
   ```bash
   ufw default deny incoming
   ufw allow 22/tcp  # 或修改后的SSH端口
   ufw allow 1883/tcp
   ufw allow 3000/tcp
   ufw allow 3001/tcp
   ufw enable
   ```
