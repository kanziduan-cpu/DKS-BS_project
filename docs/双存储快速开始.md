# SQLite + Supabase 双存储快速开始指南

## 🚀 5分钟快速部署

### 步骤 1: 配置 Supabase（2分钟）

1. 访问 [supabase.com](https://supabase.com) 创建免费项目
2. 在 SQL Editor 执行 `create-supabase-tables.sql`
3. 获取 URL 和 Key，填入 `.env` 文件

### 步骤 2: 安装依赖（1分钟）

```bash
cd cloud-server
npm install
```

### 步骤 3: 配置环境变量（1分钟）

```bash
# 编辑 .env 文件
nano .env

# 填入你的配置
SUPABASE_URL=https://xxxxxxxxxxxxx.supabase.co
SUPABASE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
STORAGE_MODE=dual
```

### 步骤 4: 启动服务（1分钟）

```bash
# 运行初始化脚本
./init-dual-storage.sh

# 启动服务
npm start

# 或使用 PM2（推荐）
pm2 start server-dual-storage.js --name warehouse-server
pm2 save
```

### 步骤 5: 测试验证（可选）

```bash
# 运行测试
node test-dual-storage.js

# 查看同步状态
node sync-monitor.js stats

# 健康检查
curl http://localhost:3000/health
```

## ⚠️ 常见问题

### Q: Supabase 连接失败？
A: 检查 `.env` 中的 URL 和 Key 是否正确，确认网络连接

### Q: 数据未同步？
A: 运行 `node sync-monitor.js stats` 查看状态，手动触发 `curl -X POST http://localhost:3000/api/sync/now`

### Q: 如何回退到单存储？
A: 修改 `.env` 中 `STORAGE_MODE=local`，使用原来的 `server.js`
