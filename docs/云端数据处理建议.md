# 云端数据处理建议

## 🤔 是否需要删除云端数据？

**简短回答：** 不需要删除，但可以清理不必要的服务。

---

## 📊 当前云端服务分析

根据你的阿里云服务器信息，你可能已经部署了以下服务：

| 服务 | 端口 | 状态 | 建议 |
|------|------|------|------|
| EMQX Dashboard | 8000 | 可能运行 | 可保留或停止 |
| EMQX MQTT Broker | 1883/8883 | 可能运行 | 可保留备用 |
| Node.js Backend | 8001 | 可能运行 | 可停止或重新配置 |
| Python Gateway | 8002 | 可能运行 | 建议停止（毕设不需要） |

## 🎯 推荐方案：共存部署

### 方案A：完全替换（推荐用于毕设）

**优势：** 服务器资源占用少，部署简单，避免端口冲突

**操作步骤：**

1. **停止现有服务**
   ```bash
   # SSH连接到服务器
   ssh root@43.99.24.178

   # 查看当前运行的服务
   ps aux | grep -E 'emqx|node|python'
   netstat -tlnp | grep -E '1883|8000|8001|8002'

   # 停止EMQX（如果运行）
   systemctl stop emqx
   systemctl disable emqx

   # 停止Python网关
   pkill -f mqtt_bridge.py

   # 停止其他Node.js服务
   pm2 stop all
   pm2 delete all
   ```

2. **删除旧代码和配置（可选）**
   ```bash
   # 备份（安全起见）
   mkdir -p /backup/old_services
   mv /opt/your-old-project /backup/old_services/

   # 或者直接删除
   rm -rf /opt/your-old-project
   ```

3. **部署新方案**
   ```bash
   # 创建新目录
   mkdir -p /opt/warehouse-monitor
   cd /opt/warehouse-monitor

   # 上传新代码
   # scp cloud-server/* root@43.99.24.178:/opt/warehouse-monitor/

   # 安装依赖
   npm install

   # 启动新服务
   pm2 start server.js --name warehouse-server
   pm2 save
   ```

---

### 方案B：共存部署（保留EMQX）

**优势：** 保留现有服务，可随时切换

**适用场景：**
- 现有服务有重要数据需要保留
- 想对比两种方案的性能
- 为论文提供"可扩展性"的演示素材

**操作步骤：**

1. **修改新方案的端口（避免冲突）**
   ```bash
   # 编辑配置文件
   nano /opt/warehouse-monitor/config.json
   ```

   修改为：
   ```json
   {
     "mqtt": {
       "port": 1884  // 改为1884，避免与EMQX冲突
     },
     "http": {
       "port": 3000  // 保持不变
     },
     "websocket": {
       "port": 3001  // 保持不变
     }
   }
   ```

2. **同时运行两个服务**
   ```bash
   # EMQX继续运行（使用1883端口）
   # 新方案运行（使用1884端口）

   # 启动新服务
   pm2 start server.js --name warehouse-server
   ```

3. **测试时选择连接方案**
   ```bash
   # 测试EMQX方案
   MQTT Broker: 43.99.24.178:1883

   # 测试新方案
   MQTT Broker: 43.99.24.178:1884
   ```

**缺点：**
- 端口管理复杂
- 资源占用增加
- 容易混淆

---

### 方案C：完全保留，临时切换

**适用场景：**
- 现有服务非常重要
- 需要快速切换回原方案

**操作步骤：**

1. **保存现有服务状态**
   ```bash
   # 查看现有服务配置
   pm2 list
   pm2 save

   # 备份现有代码
   tar -czf /backup/old-code-$(date +%Y%m%d).tar.gz /opt/your-old-project
   ```

2. **临时停止现有服务**
   ```bash
   # 停止但不删除
   pm2 stop all
   ```

3. **部署新方案测试**
   ```bash
   # 部署新方案
   cd /opt/warehouse-monitor
   npm install
   pm2 start server.js --name warehouse-server
   ```

4. **如果需要回退**
   ```bash
   # 停止新服务
   pm2 stop warehouse-server
   pm2 delete warehouse-server

   # 恢复旧服务
   pm2 restore
   ```

## 🔍 检查云端现状

在执行任何操作前，先检查服务器当前状态：

```bash
# 1. 查看运行的服务
pm2 list
systemctl list-units --type=service --state=running

# 2. 查看端口占用
netstat -tlnp | grep -E '1883|8000|8001|8002|3000|3001'

# 3. 查看磁盘使用
df -h

# 4. 查看内存使用
free -h

# 5. 查看CPU使用
top -n 1 | head -20
```

## 🗑️ 数据清理建议

### 需要清理的内容：

1. **临时日志文件**
   ```bash
   # 清理PM2日志
   pm2 flush

   # 清理系统日志
   journalctl --vacuum-time=7d

   # 清理旧日志
   find /var/log -name "*.log" -mtime +30 -delete
   ```

2. **无用的数据库文件**
   ```bash
   # 查找数据库文件
   find /opt -name "*.db" -o -name "*.sqlite"

   # 删除测试数据库（确认无用后）
   rm /path/to/unused.db
   ```

3. **临时缓存**
   ```bash
   # 清理npm缓存
   npm cache clean --force

   # 清理apt缓存
   apt-get clean
   ```

### 需要保留的内容：

1. **配置文件**（如有重要配置）
2. **数据库文件**（如有重要数据）
3. **证书文件**（如SSL证书）
4. **用户数据**（如设备注册信息）

## ⚠️ 重要提示

### 删除前的安全检查清单：

- [ ] 是否有重要数据需要备份？
- [ ] 是否有正在运行的服务会受影响？
- [ ] 是否有其他用户依赖这些服务？
- [ ] 是否已保存配置文件？
- [ ] 是否知道如何恢复服务？

### 备份命令：

```bash
# 创建完整备份
mkdir -p /backup/$(date +%Y%m%d)
tar -czf /backup/$(date +%Y%m%d)/full-backup.tar.gz \
    /opt/* \
    /etc/* \
    ~/.pm2 \
    --exclude=/backup/*
```

## 🎯 最终建议

1. ✅ 简单直接，避免混淆
2. ✅ 资源占用少，性能好
3. ✅ 端口配置简单（1883、3000、3001）
4. ✅ 毕设答辩时演示清晰
5. ✅ 符合论文"轻量级一体化"描述

### 具体操作：

# 1. 连接服务器
ssh root@43.99.24.178

## 📞 如果不确定

如果不确定是否应该删除，建议：

1. **先备份所有数据**
2. **停止但不删除**现有服务
3. **部署新方案测试**
4. **确认新方案稳定后**再决定是否删除旧方案

这样最安全，有问题可以随时回退！
