# 服务器部署快速指南

## 当前情况
你的服务器: 120.55.113.226 (Ubuntu 22.04)
正在上传到: /opt/warehouse-monitor

## 问题分析
❌ 你正在上传 `node_modules/` 目录，这包含成千上万的文件
✅ 应该在服务器上执行 `npm install` 来生成 node_modules

## 正确的部署步骤

### 方法一：先排除 node_modules，然后上传（推荐）

#### 1. 上传核心文件（排除 node_modules）

在 Windows PowerShell 中执行：

```powershell
# 进入项目目录
cd C:\Users\TSBJ\Documents\BS_project

# 使用排除模式上传（只上传必要文件）
scp -r cloud-server/node_modules/ root@120.55.113.226:/opt/warehouse-monitor/

# 上传其他文件（快速）
scp cloud-server/*.js cloud-server/*.json cloud-server/*.md cloud-server/*.sh cloud-server/*.sql root@120.55.113.226:/opt/warehouse-monitor/
```

或者更简单的方法，创建一个临时目录：

```powershell
# 创建临时目录
mkdir C:\temp-deploy
xcopy C:\Users\TSBJ\Documents\BS_project\cloud-server C:\temp-deploy /E /I /EXCLUDE:C:\temp-deploy\node_modules\ /EXCLUDE:C:\temp-deploy\data\ /EXCLUDE:C:\temp-deploy\.env

# 上传临时目录
scp -r C:\temp-deploy\* root@120.55.113.226:/opt/warehouse-monitor/

# 清理临时目录
rmdir /s /q C:\temp-deploy
```

#### 2. SSH 连接到服务器

```bash
ssh root@120.55.113.226
```

#### 3. 在服务器上安装依赖

```bash
# 进入项目目录
cd /opt/warehouse-monitor

# 安装依赖（这会生成 node_modules）
npm install

# 配置环境变量
cp .env.example .env
nano .env  # 编辑配置

# 运行初始化脚本
chmod +x init-dual-storage.sh
./init-dual-storage.sh

# 启动服务
pm2 start server-dual-storage.js --name warehouse-server
pm2 save
```

### 方法二：使用 rsync（更高效，但需要安装）

在 Windows 上安装 Git for Windows，它包含 rsync：

```powershell
# 使用 rsync 排除 node_modules
rsync -avz --exclude='node_modules' --exclude='data' --exclude='.env' \
  cloud-server/ root@120.55.113.226:/opt/warehouse-monitor/
```

---

### 方法三：直接在服务器上克隆（最简单）

#### 1. 在服务器上克隆仓库

```bash
ssh root@120.55.113.226

# 安装 git（如果没有）
apt-get update
apt-get install -y git

# 克隆你的 GitHub 仓库
cd /opt
git clone https://github.com/kanziduan-cpu/DKS-BS_project.git warehouse-monitor
cd warehouse-monitor/cloud-server
```

#### 2. 安装依赖并启动

```bash
# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
nano .env  # 填入你的配置

# 初始化
chmod +x init-dual-storage.sh
./init-dual-storage.sh

# 启动服务
pm2 start server-dual-storage.js --name warehouse-server
pm2 save
```

---

### 方法四：上传 .tar.gz 压缩包（推荐用于大项目）

#### 1. 在本地打包

```powershell
cd C:\Users\TSBJ\Documents\BS_project

# 创建压缩包（排除 node_modules）
tar --exclude='node_modules' --exclude='data' --exclude='.env' -czf warehouse-monitor.tar.gz cloud-server/
```

#### 2. 上传压缩包

```powershell
scp warehouse-monitor.tar.gz root@120.55.113.226:/opt/
```

#### 3. 在服务器上解压

```bash
ssh root@120.55.113.226

cd /opt
rm -rf warehouse-monitor  # 删除旧目录
mkdir warehouse-monitor
tar -xzf warehouse-monitor.tar.gz -C warehouse-monitor --strip-components=1
rm warehouse-monitor.tar.gz

cd warehouse-monitor
npm install  # 在服务器上安装依赖
```
