# 🔍 Logcat 日志分析指南

## 📊 如何过滤重要日志

您刚才看到的日志主要是系统级错误，需要过滤掉。

### 在Logcat中使用精确过滤器

在Logcat搜索框中输入以下内容来查看应用相关日志：

```
tag:WarehouseMonitor
```

或者查看特定模块：

```
tag:WarehouseMonitor_Network
tag:WarehouseMonitor_Database
tag:WarehouseMonitor_MQTT
tag:HttpInterceptor
```

### 🚫 如何忽略系统日志

系统级日志（您可以忽略）：
- `com.google.android.gms` - Google服务日志
- `com.android.networkstack` - 网络栈日志
- `TrafficStats` - 流量统计
- `NetworkMonitor` - 网络监控
- `System.out` - 系统输出（除非看到应用相关内容）

### ✅ 您应该关注的日志

应用相关日志（重要）：
- `WarehouseMonitor_*` - 您的应用的所有日志
- `HttpInterceptor` - HTTP请求和响应
- `MqttManager` - MQTT连接状态
- `DatabaseLogger` - 数据库操作

## 🎯 如何查看应用日志

### 方法1：使用精确过滤器

在Logcat搜索框中输入：

```
tag:WarehouseMonitor
```

### 方法2：查看特定包

```
package:com.warehouse.monitor
```

### 方法3：使用ADB命令

```bash
# 查看应用日志
adb logcat -s WarehouseMonitor:*:V

# 或者查看所有包
adb logcat | findstr "warehouse"
```

## 🔧 故障排查

### 如果仍然看不到应用日志

1. **确保应用正在运行**
   - 检查应用是否已启动
   - 执行一些操作（如登录、加载数据）

2. **检查应用是否真的生成了日志**
   - 查看代码中是否有 `AppLogger.debug()` 等日志调用
   - 确保应用正在执行相关操作

3. **清除并重新捕获日志**
   ```bash
   adb logcat -c
   adb logcat -s WarehouseMonitor:*:V
   ```

4. **检查应用包名**
   - 在AndroidManifest.xml中确认包名
   - 默认应该是 `com.warehouse.monitor`

## 🎯 调试建议

### 1. 启动应用后立即查看日志

```bash
# 清除旧日志
adb logcat -c

# 实时查看应用日志
adb logcat -s WarehouseMonitor:*:V
```

### 2. 在应用中执行操作

- 打开应用
- 尝试登录
- 查看仓库列表
- 操作设备

观察Logcat中的输出。

### 3. 如果看到错误，按以下步骤排查

1. **网络错误**：
   - 检查服务器是否在线：`http://120.55.113.226:3001`
   - 检查网络连接
   - 检查防火墙设置

2. **数据库错误**：
   - 查看AppDatabase.java中的表结构
   - 检查数据类型是否匹配
   - 验证数据库版本迁移

3. **MQTT错误**：
   - 检查MQTT服务器地址：`mqtt://120.55.113.226:1883`
   - 检查MQTT用户名和密码
   - 检查网络端口是否开放

## 📝 如何保存重要日志

### 方法1：Logcat导出
- Logcat右上角 ⋮ → Export to File
- 保存为文本文件

### 方法2：ADB保存
```bash
# 保存所有应用日志
adb logcat -s WarehouseMonitor:* > app_logs.txt

# 实时保存到文件
adb logcat -s WarehouseMonitor:* | tee app_logs.txt
```

### 方法3：复制粘贴
- 在Logcat中选择相关日志行
- Ctrl+C 复制
- 粘贴到文本编辑器

## 📊 日志过滤技巧

### 组合过滤器

```
# 只看错误日志
tag:WarehouseMonitor level:error

# 只看网络和MQTT日志
tag:WarehouseMonitor_Network | tag:WarehouseMonitor_MQTT

# 排除系统日志
-tag:com.google.android.gms -tag:com.android.networkstack
```

### 正则表达式

```
# 搜索所有网络相关日志
.*Network.*

# 搜索特定错误模式
.*Connection.*failed
```
