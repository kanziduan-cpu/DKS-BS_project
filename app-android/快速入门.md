# 🚀 WarehouseMonitor 快速入门指南

## 📍 Logcat查看日志的位置

### 🖥️ 在Android Studio中查看（最简单）

1. **打开Logcat窗口**
   ```
   方法1：点击底部 "Logcat" 标签
   方法2：快捷键 Alt + 6
   方法3：View → Tool Windows → Logcat
   ```

2. **选择您的设备**
   - 在Logcat顶部点击设备选择器
   - 选择真机或模拟器

3. **输入过滤器**
   ```
   在搜索框中输入：tag:WarehouseMonitor
   ```
   这样就能看到应用的所有日志！

### 🔍 快速过滤示例

| 想看什么 | 输入过滤器 |
|----------|------------|
| 所有日志 | `tag:WarehouseMonitor` |
| 网络日志 | `tag:WarehouseMonitor_Network` |
| 数据库日志 | `tag:WarehouseMonitor_Database` |
| MQTT日志 | `tag:WarehouseMonitor_MQTT` |
| HTTP请求 | `tag:HttpInterceptor` |
| 只看错误 | `*:E` |

### 💡 小技巧
- 点击Logcat右上角的搜索图标可以搜索特定关键词
- 改变日志级别可以看到不同详细程度的信息
- 选择"Verbose"可以看到最详细的日志

## 📱 ADB命令行查看

### 查看所有日志
```bash
adb logcat -s WarehouseMonitor:*:V
```

### 查看特定模块
```bash
# 网络日志
adb logcat -s WarehouseMonitor_Network:*:V

# 数据库日志
adb logcat -s WarehouseMonitor_Database:*:V

# MQTT日志
adb logcat -s WarehouseMonitor_MQTT:*:V
```

### 清除旧日志
```bash
adb logcat -c
```

## ❓ 常见问题

### Q1: 为什么看不到日志？
**A:**
1. 检查设备是否连接：`adb devices`
2. 确保应用正在运行
3. 清除旧日志：`adb logcat -c`
4. 检查Logcat的设备选择器

### Q2: 日志太多看不过来怎么办？
**A:**
1. 使用精确的标签过滤：`tag:WarehouseMonitor_Network`
2. 选择更高的日志级别：`Info` 或 `Warn`
3. 使用搜索功能查找关键信息
4. 使用自动化脚本过滤

### Q3: 中文日志显示乱码？
**A:**
在Logcat窗口右键 → Change Encoding → UTF-8

### Q4: 想保存日志到文件？
**A:**
在Android Studio中：Logcat右上角 ⋮ → Export to File

或者使用命令：
```bash
adb logcat -s WarehouseMonitor:* > app_logs.txt
```
