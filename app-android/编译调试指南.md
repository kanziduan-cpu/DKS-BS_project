## 编译步骤

### 1. 在Android Studio中打开项目
```bash
# 打开Android Studio
# File -> Open -> 选择 C:\Users\TSBJ\Documents\BS_project\app-android 目录
```

### 2. 同步Gradle项目
- 打开项目后，Android Studio会自动同步Gradle
- 或者点击：File -> Sync Project with Gradle Files

### 3. 清理项目（如果有之前的构建）
- Build -> Clean Project

### 4. 编译项目
- Build -> Rebuild Project

### 5. 选择构建变体
- View -> Tool Windows -> Build Variants
- 选择 `debug` 变体进行调试
- 选择 `release` 变体进行发布

## 调试配置

### 1. 连接设备
- **真机调试**：
  - 在手机上开启开发者选项
  - 启用USB调试
  - 用USB连接电脑
  - Android Studio会自动识别设备

- **模拟器调试**：
  - Tools -> Device Manager
  - 创建新的虚拟设备（推荐Pixel系列）
  - 选择API 26+的系统镜像

### 2. 运行应用
- 点击工具栏上的运行按钮（绿色三角形）
- 或者按 Shift + F10

### 3. 调试应用
- 点击工具栏上的调试按钮（带虫子的绿色三角形）
- 或者按 Shift + F9

### 4. 查看日志
- View -> Tool Windows -> Logcat
- 过滤器设置：WarehouseMonitor

## 日志系统

项目已集成完整的日志系统，使用以下过滤标签：

### 主要日志标签
- `WarehouseMonitor_Network` - 网络连接日志
- `WarehouseMonitor_Database` - 数据库操作日志
- `WarehouseMonitor_MQTT` - MQTT连接日志
- `WarehouseMonitor_Business` - 业务逻辑日志
- `HttpInterceptor` - HTTP请求拦截器日志

### 日志过滤
在Logcat中使用以下过滤器：
```
tag:WarehouseMonitor
```

或者查看特定模块：
```
tag:WarehouseMonitor_Network
```

## 常见问题解决

### 1. Gradle同步失败
- 检查网络连接
- 尝试使用阿里云Maven镜像（已配置）
- File -> Invalidate Caches / Restart

### 2. 编译错误
- Build -> Clean Project
- 删除 .gradle 目录
- 重新同步项目

### 3. 设备无法连接
- 检查USB调试是否开启
- 尝试更换USB端口
- 检查设备授权对话框

### 4. 应用无法启动
- 查看Logcat日志
- 检查AndroidManifest.xml配置
- 验证所有Activity都已注册

## 调试技巧

### 1. 断点调试
- 在代码行号左侧点击设置断点
- 调试模式下运行应用
- 应用会在断点处暂停

### 2. 日志输出
- 使用 `AppLogger.debug("Module", "message")` 输出调试信息
- 在Logcat中实时查看

### 3. 网络请求监控
- OkHttp拦截器会自动记录所有HTTP请求
- 查看请求和响应的详细信息

### 4. 数据库查询监控
- 数据库操作会自动记录日志
- 查看查询性能和结果
