# Logcat 日志查看说明

## 🔍 Logcat 快速入门

### 在Android Studio中查看（推荐）

1. **打开Logcat窗口**
   - 点击底部 `Logcat` 标签
   - 快捷键：`Alt + 6`
   - 菜单：`View` → `Tool Windows` → `Logcat`

2. **选择设备**
   - 在设备选择器中选择您的设备
   - 点击绿色的播放按钮开始捕获日志

3. **设置过滤器**

#### 标签过滤器（推荐使用）
```
tag:WarehouseMonitor
```

#### 按模块过滤
```
tag:WarehouseMonitor_Network     # 网络日志
tag:WarehouseMonitor_Database    # 数据库日志
tag:WarehouseMonitor_MQTT        # MQTT日志
tag:WarehouseMonitor_Business     # 业务日志
```

#### HTTP请求日志
```
tag:HttpInterceptor
```

#### 组合过滤
```
tag:WarehouseMonitor | tag:HttpInterceptor
```

4. **日志级别选择**
   - `Verbose` - 所有日志（调试用）
   - `Debug` - 调试及以上
   - `Info` - 信息及以上
   - `Warn` - 警告及以上
   - `Error` - 仅错误

## 🖥️ 命令行日志查看

### 使用ADB命令

#### 查看所有日志
```bash
adb logcat
```

#### 查看应用日志
```bash
adb logcat -s WarehouseMonitor:*:V
```

#### 查看特定模块日志
```bash
# 网络日志
adb logcat -s WarehouseMonitor_Network:*:V

# 数据库日志
adb logcat -s WarehouseMonitor_Database:*:V

# MQTT日志
adb logcat -s WarehouseMonitor_MQTT:*:V
```

#### 清除日志缓冲区
```bash
adb logcat -c
```

#### 保存日志到文件
```bash
adb logcat -s WarehouseMonitor:* > app_logs.txt
```

## 📋 日志标签说明

### 应用日志标签
| 标签 | 说明 | 日志内容 |
|------|------|----------|
| `WarehouseMonitor_Network` | 网络连接 | HTTP请求状态、响应时间、网络错误 |
| `WarehouseMonitor_Database` | 数据库操作 | CRUD操作、查询性能、数据库错误 |
| `WarehouseMonitor_MQTT` | MQTT连接 | 连接状态、消息收发、订阅主题 |
| `WarehouseMonitor_Business` | 业务逻辑 | 用户操作、功能流程 |

### 系统日志标签
| 标签 | 说明 | 日志内容 |
|------|------|----------|
| `HttpInterceptor` | HTTP拦截器 | 请求详情、响应内容、耗时统计 |
| `MqttManager` | MQTT管理器 | 连接过程、订阅状态、消息处理 |
| `DatabaseLogger` | 数据库日志 | 初始化状态、操作记录、性能监控 |

## 🔍 日志搜索技巧

### 在Android Studio中搜索
1. 在Logcat窗口中，点击搜索图标 🔍
2. 输入关键词进行搜索：
   - `error` - 搜索错误
   - `timeout` - 搜索超时
   - `MQTT连接成功` - 搜索特定消息
   - `120.55.113.226` - 搜索服务器IP

### 使用正则表达式
```
# 搜索所有网络错误
.*Network.*error

# 搜索所有MQTT状态
.*MQTT.*状态

# 搜索特定端点
.*3001/api/.*
```

## ⚠️ 常见问题

### 问题1：看不到日志
**解决方法：**
1. 检查设备是否连接：`adb devices`
2. 确保应用正在运行
3. 尝试清除日志缓冲区：`adb logcat -c`
4. 检查Logcat的设备选择器

### 问题2：日志太多看不过来
**解决方法：**
1. 使用精确的标签过滤：`tag:WarehouseMonitor_Network`
2. 提高日志级别选择：选择 `Info` 或 `Warn`
3. 使用搜索功能查找关键信息
4. 清除旧日志后重新捕获

### 问题3：Logcat窗口卡住
**解决方法：**
1. 停止日志捕获
2. 清除日志缓冲区：`adb logcat -c`
3. 重启Logcat窗口
4. 或者重启Android Studio

### 问题4：中文日志显示乱码
**解决方法：**
1. Logcat窗口右键 → Change Encoding → UTF-8
2. 确保系统编码设置为UTF-8

## 📝 日志导出

### 导出到文件
在Android Studio中：
1. Logcat窗口右上角 → ⋮ → Export to File
2. 选择保存位置
3. 命名：`warehouse_logs_<时间>.txt`

### 使用ADB导出
```bash
# 实时保存
adb logcat -s WarehouseMonitor:* > app_logs.txt

# 清空后保存
adb logcat -c && adb logcat -s WarehouseMonitor:* > app_logs.txt

# 带时间戳保存
adb logcat -s WarehouseMonitor:* > app_logs_%date:~0,10%_%time:~0,8%.txt
```

## 🛠️ 高级技巧

### 1. 条件断点 + 日志
- 设置断点
- 右键断点 → Log evaluated expression
- 输入：`AppLogger.debug("Debug", "变量值 = " + variable)`
- 可以在断点时自动输出日志

### 2. 日志颜色配置
- File → Settings → Editor → Color Scheme → Logcat
- 为不同级别设置不同颜色
- 提高日志可读性

### 3. 自定义日志过滤器
在 `logcat-filter` 文件中创建常用过滤器：
```
# my-filters.conf
tag:WarehouseMonitor.*level:error
tag:WarehouseMonitor.*level:warn
```

### 4. 实时日志分析
使用脚本监控特定模式：
```bash
adb logcat -s WarehouseMonitor_Network | find "error"
```

## 📱 设备连接检查

### 检查ADB连接
```bash
adb devices
```

期望输出：
```
List of devices attached
<设备ID>    device
```

### 如果设备未识别
1. 检查USB调试是否开启
2. 更换USB端口
3. 授权USB调试（手机上弹出对话框）
4. 重启ADB服务：`adb kill-server && adb start-server`

5. **使用描述性的日志消息**
6. **包含时间戳和上下文信息**
7. **记录关键路径的日志**
