# 🧪 应用与云端通信测试指南

## 📊 通信架构

```
Android App
    ├─ HTTP API (3001) ←→ 服务器API服务
    │   ├─ 登录认证
    │   ├─ 数据查询
    │   └─ 设备控制
    │
    └─ MQTT (1883) ←→ 服务器MQTT服务
        ├─ 传感器数据
        ├─ 设备状态
        └─ 实时消息
```

## ✅ 测试准备

### 1. 服务器检查

```bash
# SSH连接到服务器
ssh root@120.55.113.226

# 检查API服务（端口3001）
netstat -tlnp | grep 3001

# 检查MQTT服务（端口1883）
netstat -tlnp | grep 1883
# 如果没有输出，需要安装MQTT：./install_mqtt.sh

# 检查服务状态
systemctl status warehouse-monitor  # API服务
systemctl status mosquitto          # MQTT服务
```

### 2. Android应用配置

确认配置文件中的服务器地址：
- **API地址**：`http://120.55.113.226:3001/api/`
- **MQTT地址**：`mqtt://120.55.113.226:1883`

## 🧪 测试步骤

### 测试1：HTTP API连接

#### 方法1：使用浏览器测试
```
打开浏览器访问：
http://120.55.113.226:3001/api/
```

期望返回：API服务正常响应（可能是404或错误信息，说明服务在线）

#### 方法2：使用curl测试
```bash
# 测试API根路径
curl http://120.55.113.226:3001/api/

# 测试登录接口
curl -X POST http://120.55.113.226:3001/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456"}'
```

#### 方法3：Android应用测试
1. 启动应用
2. 尝试登录
3. 在Logcat中查看日志：
   ```
   tag:WarehouseMonitor_Network
   ```

期望看到：
```
发送请求: http://120.55.113.226:3001/api/login
收到响应: 200 OK
网络请求状态 - 端点: /api/login, 耗时: xxxms, 状态: 成功
```

### 测试2：MQTT连接

#### 方法1：使用MQTT客户端测试

在服务器上打开终端：
```bash
# 订阅主题
mosquitto_sub -h localhost -p 1883 -u testuser -P 123456 -t "sensor/#" -v
```

在另一个终端发布消息：
```bash
# 发布测试数据
mosquitto_pub -h localhost -p 1883 -u testuser -P 123456 \
  -t "sensor/data" \
  -m '{"temperature":25.5,"humidity":60.2,"timestamp":1678901234567}'
```

#### 方法2：Android应用测试

1. 启动应用
2. 在Logcat中查看日志：
   ```
   tag:WarehouseMonitor_MQTT
   ```

期望看到：
```
MQTT连接成功
订阅主题成功: sensor/data
订阅主题成功: sensor/status
```

如果看到 `MQTT未连接`，说明MQTT服务未运行。

## ⚠️ 常见问题

### 问题1：HTTP API连接失败
**症状**：应用无法登录或获取数据

**检查**：
1. 服务器API服务是否运行：`systemctl status warehouse-monitor`
2. 端口3001是否开放：`netstat -tlnp | grep 3001`
3. 防火墙是否允许：`ufw status`

**解决**：
```bash
# 重启API服务
systemctl restart warehouse-monitor

# 开放防火墙
ufw allow 3001/tcp
```

### 问题2：MQTT未连接
**症状**：Logcat显示"MQTT未连接"

**检查**：
1. MQTT服务是否安装：`systemctl status mosquitto`
2. 端口1883是否开放：`netstat -tlnp | grep 1883`

**解决**：
```bash
# 安装MQTT服务
./install_mqtt.sh

# 或手动安装
apt install -y mosquitto mosquitto-clients
systemctl start mosquitto

# 开放防火墙
ufw allow 1883/tcp
```

### 问题3：网络超时
**症状**：连接超时或长时间无响应

**检查**：
1. 网络连接：`ping 120.55.113.226`
2. 端口连通性：`telnet 120.55.113.226 3001`

**解决**：
- 检查网络连接
- 检查防火墙规则
- 检查服务是否正常运行
