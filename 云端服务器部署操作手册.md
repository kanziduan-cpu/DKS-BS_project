## 🚀 部署步骤

### 方案一：使用 PowerShell 自动部署脚本（推荐）

#### 前置要求

1. **安装 OpenSSH 客户端**
   - Windows 10/11 已内置
   - 如需安装：`设置 > 应用 > 可选功能 > 添加功能 > OpenSSH 客户端`

2. **配置 SSH 密钥（可选）**
   ```powershell
   # 生成 SSH 密钥对
   ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
   
   # 将公钥上传到服务器
   type $env:USERPROFILE\.ssh\id_rsa.pub | ssh root@120.55.113.226 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
   ```

#### 执行部署

1. **打开 PowerShell**，右键点击项目根目录的 `upload-to-server.ps1`
2. **选择 "使用 PowerShell 运行"**
3. **等待部署完成**

脚本会自动完成：
- ✅ 上传项目文件到服务器
- ✅ 安装 Node.js 依赖
- ✅ 创建配置文件

#### 手动配置

部署完成后，需要手动配置：

```bash
# 1. 登录服务器
ssh root@120.55.113.226

# 2. 进入项目目录
cd /opt/warehouse-monitor/cloud-server

# 3. 创建环境变量文件
nano .env
```

添加以下内容：
```env
# 服务器配置
PORT=3000
NODE_ENV=production

# 数据库配置（可选，如果使用 Supabase）
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_supabase_anon_key

# MQTT 配置
MQTT_HOST=localhost
MQTT_PORT=1884

# 日志级别
LOG_LEVEL=info
```

保存退出：`Ctrl + O`, `Enter`, `Ctrl + X`

### 方案二：手动部署

#### 1. 上传项目文件

```bash
# 使用 scp 上传
scp -r cloud-server/ root@120.55.113.226:/opt/warehouse-monitor/

# 或者使用 rsync（推荐，支持断点续传）
rsync -avz --progress cloud-server/ root@120.55.113.226:/opt/warehouse-monitor/cloud-server/
```

#### 2. 安装依赖

```bash
# SSH 登录服务器
ssh root@120.55.113.226

# 进入项目目录
cd /opt/warehouse-monitor/cloud-server

# 安装生产环境依赖
npm install --production
```

#### 3. 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑配置
nano .env
```

配置内容同上。

#### 4. 启动服务

##### 使用 PM2（推荐）

```bash
# 全局安装 PM2
npm install -g pm2

# 启动服务
pm2 start server.js --name warehouse-api

# 查看状态
pm2 status

# 设置开机自启
pm2 startup
pm2 save

# 查看日志
pm2 logs warehouse-api
```

##### 使用 systemd

```bash
# 创建服务文件
nano /etc/systemd/system/warehouse-api.service
```

添加以下内容：
```ini
[Unit]
Description=Warehouse Monitor API Server
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/warehouse-monitor/cloud-server
ExecStart=/usr/bin/node server.js
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
# 重载 systemd
systemctl daemon-reload

# 启动服务
systemctl start warehouse-api

# 设置开机自启
systemctl enable warehouse-api

# 查看状态
systemctl status warehouse-api

# 查看日志
journalctl -u warehouse-api -f
```

##### 直接运行（测试用）

```bash
cd /opt/warehouse-monitor/cloud-server
node server.js
```

## 🔒 防火墙配置

### 检查防火墙状态

```bash
# UFW (Ubuntu)
ufw status

# firewalld (CentOS)
firewall-cmd --list-all
```

### 开放端口

```bash
# Ubuntu (UFW)
ufw allow 3000/tcp    # API 服务
ufw allow 1884/tcp    # MQTT 服务
ufw allow 3001/tcp    # WebSocket 服务
ufw reload

# CentOS (firewalld)
firewall-cmd --zone=public --add-port=3000/tcp --permanent
firewall-cmd --zone=public --add-port=1884/tcp --permanent
firewall-cmd --zone=public --add-port=3001/tcp --permanent
firewall-cmd --reload
```

### 阿里云安全组

登录阿里云控制台：
1. 进入 **云服务器 ECS** 实例
2. 点击 **安全组** > **配置规则**
3. 添加入方向规则：

| 协议类型 | 端口范围 | 授权对象 | 描述 |
|---------|----------|----------|------|
| TCP | 3000/3000 | 0.0.0.0/0 | API 服务 |
| TCP | 1884/1884 | 0.0.0.0/0 | MQTT 服务 |
| TCP | 3001/3001 | 0.0.0.0/0 | WebSocket |

## 🔧 故障排查

### 问题 1：服务无法启动

**检查点：**
```bash
# 1. 检查端口是否被占用
netstat -tlnp | grep 3000

# 2. 检查依赖是否安装
cd /opt/warehouse-monitor/cloud-server
npm list

# 3. 查看详细错误
node server.js
```

**解决方案：**
```bash
# 杀死占用端口的进程
kill -9 $(lsof -t -i:3000)

# 重新安装依赖
rm -rf node_modules
npm install --production
```

### 问题 2：数据库初始化失败

**检查点：**
```bash
# 检查 data 目录权限
ls -la /opt/warehouse-monitor/cloud-server/data/

# 检查磁盘空间
df -h
```

**解决方案：**
```bash
# 创建数据目录
mkdir -p data
chmod 755 data
```

### 问题 3：无法从外网访问

**检查点：**
1. 防火墙是否开放端口
2. 阿里云安全组是否配置
3. 服务是否监听 0.0.0.0（所有地址）

**解决方案：**
```bash
# 检查监听地址
netstat -tlnp | grep 3000

# 应该显示 0.0.0.0:3000 而不是 127.0.0.1:3000
```

### 问题 4：MQTT 连接失败

**检查点：**
```bash
# 检查 MQTT 服务状态
systemctl status mosquitto

# 检查 MQTT 配置
cat /etc/mosquitto/mosquitto.conf

# 测试 MQTT 端口
telnet 120.55.113.226 1884
```

## 📈 性能优化

### 1. 使用 PM2 集群模式

```bash
# 根据CPU核心数启动多个进程
pm2 start server.js --name warehouse-api -i max
```

### 2. 启用 Gzip 压缩

在 `server.js` 中添加：
```javascript
const compression = require('compression');
app.use(compression());
```

### 3. 数据库连接池优化

使用连接池管理数据库连接。

### 4. 日志轮转

配置 PM2 日志轮转：
```bash
pm2 install pm2-logrotate
pm2 set pm2-logrotate:max_size 10M
pm2 set pm2-logrotate:retain 7
```
